# Contributing

Thank you for considering contributing to Hermes Memory Vault.

## Goals
- Keep the project lightweight.
- Preserve readability of raw notes.
- Avoid unnecessary dependencies.
- Keep the MCP interface small and stable.

## Suggested workflow
1. Fork the repository.
2. Create a feature branch.
3. Make focused changes.
4. Test the MCP server locally.
5. Submit a pull request with a clear description.

## What to include in a PR
- A short summary of the change.
- Any relevant screenshots or logs.
- Notes about compatibility or migration impact.

## Code style
- Prefer clear, explicit Python code.
- Keep functions small and readable.
- Avoid heavy abstractions unless they improve maintainability.

## Reporting issues
If you find a bug or want a feature, please include:
- What you tried
- What you expected
- What actually happened
- Relevant logs or traces
