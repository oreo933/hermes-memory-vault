Hermes Memory Vault — Architecture

A local-first external memory layer for Hermes Agent. The diagram shows how MCP connects Hermes to a lightweight memory service backed by Markdown notes and SQLite FTS5 search.

local-first · lightweight · MCP-native

What it solves

  • Hermes Agent's native memory is too small
  • Important preferences and work context no longer get lost
  • Long-term memory becomes readable, searchable, and reusable

Core stack

  • Markdown for raw notes
  • SQLite FTS5 for local search
  • MCP for Hermes tool access

Design principle

  • Local-first
  • Lightweight
  • Easy to audit
  • Easy to extend later
